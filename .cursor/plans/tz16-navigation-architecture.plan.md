# ТЗ 16 — Архитектура навигации (роли и распределение функций)

Фундамент перед ТЗ-17 (очистка дублей). Без удаления функций, только перераспределение.

## Роли
1. **Гость** — не авторизован
2. **Пользователь** — ищет жильё (основная роль)
3. **Арендодатель** — пользователь, создавший объявление (+ мои объявления, доход и т.д.)
4. **Админ** — отдельно, пункт только в меню аватара

## 1. Гость
- Доступно: поиск, просмотр объявлений, избранное (локально), сообщения → после входа, регистрация.
- **Шапка:** логотип, поиск, Войти.
- **Нижнее меню (mobile):** Главная, Поиск, Избранное, Войти.
- Кнопка «Разместить» на главной → ведёт на регистрацию/вход.

## 2. Пользователь
- Доступ: поиск, сообщения, избранное, профиль, бронирования.
- **Шапка:** логотип, поиск, избранное, уведомления, аватар.
- **Нижнее меню (mobile):** Главная, Поиск, Сообщения, Избранное, Профиль.

## 3. Арендодатель
- Тот же интерфейс, что и пользователь. Дополнительно: мои объявления, добавить объявление, бронирования гостей, доход — через профиль/кабинет. Ничего не дублируем.

## 4. Меню аватара
- Профиль, Мои объявления, Бронирования, Настройки, Выйти.
- При отсутствии объявлений — «Мои объявления» есть (пустое состояние).
- Для админа: + Админ панель.

## 5. Бургер-меню
- Только сервисные страницы: **О сервисе**, **Помощь**, **Тарифы**, **Контакты**.
- Убрать из бургера: сообщения, профиль, избранное, уведомления (и прочие дубли).

## 6. Сообщения
- Только: иконка в шапке + нижнее меню (mobile). Нигде больше.

## 7. Избранное
- Только: иконка в шапке + нижнее меню.

## 8. Уведомления
- Только: иконка в шапке (на mobile можно в нижнем меню — опционально).

## 9. Кнопка «Разместить»
- Показывать: на главной, в профиле, FAB mobile (кабинет).
- Не показывать: в бургере, в меню аватара.

## 10. Админ
- В меню аватара добавить «Админ панель» только для роли admin.

## Запрещено
- Удалять функции, ломать роуты/auth, сильно менять дизайн. Только архитектура.

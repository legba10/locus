# ТЗ 14 — Кабинет владельца (уровень Airbnb Pro / Cian Pro)

Фокус: логичный, профессиональный кабинет без перегрузки и дублей. Только реорганизация и достройка.

## 1. Принцип
Кабинет = центр управления объектами и доходом. Не соцсеть, не профиль.

## 2. Структура меню кабинета
Обзор | Объявления | Бронирования | Сообщения* | Финансы | Продвижение | Профиль | Настройки  
\* Сообщения — один вход (нижнее меню → /messages); в кабинете пункт ведёт на /messages, без дублирования контента.

Админ: + пункт «Админ» (если роль admin).

## 3. Обзор (Dashboard)
- Верхняя аналитика: Активные объявления, Бронирования сегодня, Доход месяц, Просмотры.
- Быстрые действия: Создать объявление, Продвинуть, Календарь. Без дубля «Сообщения».
- Уведомления (новый отзыв, бронь, сообщение) — по возможности.

## 4. Объявления
Вкладки: Активные | Черновики | Архив. Карточка: фото, цена, статус, просмотры, доход, редактировать. Действия: Редактировать, Продвинуть, Отключить, Удалить.

## 5. Редактор объявления
Шаги: Фото → Информация → Удобства → Цена → Бронирование → Публикация.  
Ошибка: «минимум 5 фото» не дублировать — одна формулировка.

## 6. Бронирования
Таблица: Дата, Гость, Сумма, Статус. Фильтры. Карточка: чат, детали, отменить.

## 7. Сообщения
Один центр: только нижнее меню → сообщения. В кабинете — пункт меню ведёт на /messages (редирект), контент не дублируется.

## 8. Финансы / Продвижение / Профиль / Настройки
По текущей реализации; при необходимости — выравнивание под ТЗ.

## 9. UX
- Никаких дублей: сообщения в одном месте, «Создать объявление» — только в кабинете (обзор, объявления, пустое состояние).
- Пустые состояния: нет объявлений → «Создать объявление»; нет бронирований → «Продвигайте объявление».

# ТЗ-32 — Аудит UI и фиксация архитектуры экранов

## Цель
Единая логика продукта, убрать рассинхрон, зафиксировать архитектуру (база для следующих задач).

## Статус: частично выполнено

### Выполнено

**1. Сетка и отступы (п. 2.1)**  
- В профиле введены токены: `SPACE_SECTION` (mb-10 = 40px между секциями), `SPACE_BLOCK` (24), `SECTION_TITLE` (mb-4).  
- Используется единая сетка 8px (8/16/24/32/40).

**2. Удаление дублей меню (п. 2.2)**  
- Вложенное меню профиля (ProfileSidebar) убрано: подстраницы профиля рендерятся в одну колонку с одной ссылкой «← Назад в профиль» из layout.  
- Остаются: нижнее меню (глобальная навигация), экран профиля (кабинет), выпадающее меню аватарки (быстрые действия).  
- Дублирующие ссылки «Назад в профиль» удалены со страниц: listings, bookings, finance, promo (одна точка входа в layout).

**3. Архитектура профиля (п. 3.1)**  
- Структура главной профиля приведена к ТЗ-32:  
  - [Аватар + имя] [Редактировать профиль]  
  - **Управление жильём:** Мои объявления, Бронирования, Календарь, Продвижение, Финансы (+ Стать арендодателем / Добавить объявление по роли).  
  - **Аналитика** (для арендодателя): Просмотры, Статистика.  
  - **Система:** Настройки, Безопасность, Сессии.  
  - Выйти.  
- Редиректы: `/profile/calendar` → `/profile/bookings`, `/profile/sessions` → `/profile/security`, `/profile/analytics/stats` → `/profile/analytics`.

**4. Layout подстраниц профиля**  
- `ProfileLayoutV2`: на подстраницах только контейнер + «Назад в профиль» + `children`, без сайдбара.  
- Добавлен `pb-24 md:pb-8` для учёта нижней панели на мобильных.

### Осталось (по приоритету ТЗ-32)

- **4.** Карточка объявления: режимы по роли (гость / арендодатель / админ), одна нижняя панель; на странице объявления скрывать глобальное нижнее меню и показывать панель действий.  
- **5.** Модерация: кнопка «Модерация» только при `status === 'pending'`; при active — обычный режим, при rejected — «Отклонено».  
- **6.** Отступы на странице объявления: gap между фото и текстом 16px, карта по кнопке, не «торчит».  
- **7.** Фильтры: единый bottom-sheet, категории, бюджет, комнаты, срок, AI-режим.  
- **8.** AI-подбор: привязка состояний (сохранение, поиск, уведомление).  
- **9.** Кабинет арендодателя: при необходимости отдельные доработки.

# TZ 21 — Финальная подготовка к запуску + PUSH в production

**Статус:** Release Candidate 1 (после успешного push и проверки production).

---

## Запрещено после этого ТЗ

Не менять без отдельного ТЗ/багфикса:
- hero (тексты, печать, кнопка)
- фильтры (структура, кнопки)
- layout главной (порядок блоков)
- header (иконки, профиль, «Разместить»)

Только багфиксы.

---

## Порядок блоков главной (зафиксирован)

1. Header (в layout)
2. Hero (заголовок, печатная строка, подзаголовок, CTA, AI-панель внутри)
3. Популярные города
4. Поисковая панель + Фильтр (город, бюджет, тип, «Показать варианты», «Фильтры»)
5. Актуальные объявления
6. Статистика (15K+ и т.д.) — **один раз**, после ленты
7. Рекомендации AI, Последние просмотренные, MarketAnalysisBlock, HowItWorks, AIBlock
8. Новости рынка
9. Сдать жильё / CTA
10. Футер (в layout)

Статистика нигде больше не дублируется.

---

## Проверки перед push

- [ ] Hero: печать корректна, кнопка «Подобрать жильё» работает
- [ ] Header: desktop — уведомления, тема, профиль, «Разместить»; mobile — бургер, профиль, уведомления; одна точка входа в профиль
- [ ] Фильтр: открытие, закрытие, применение, Escape; «Показать варианты» ведёт на /listings
- [ ] Карточки: открываются, фото не растягивается, цена читаема
- [ ] Страница объявления: календарь, кнопки, отступы, mobile
- [ ] Темы: light без засветов, dark без серых артефактов
- [ ] Console: нет лишних errors/warnings (допустимы console.error в catch)
- [ ] Сборка: `npm run build` без ошибок

---

## Порядок PUSH

1. `git pull`
2. `npm run build`
3. Проверить локально (mobile + desktop)
4. `git add` + commit
5. `git push`
6. Проверить Vercel (production)

---

final UI stabilization before release
```

fix: layout stabilization, header cleanup, filters UX polish
```

## Коммент для разработчика

Финальная стабилизация интерфейса перед запуском.

Задача: ничего не ломать, ничего не перестраивать.  
Только фиксы визуала, отступов, состояний и стабильности.

Проверить: главную, header, фильтры, карточки, страницу объявления, mobile + desktop, dark/light темы.

Удалить debug-код и лишние console.log. Проверить console errors.

После проверки — обычный push без изменений структуры.

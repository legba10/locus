# ТЗ-21: Полная нормализация структуры навигации и ролей (mobile + desktop)

## Цель
Привести интерфейс к логичной архитектуре, убрать дубли, стабилизировать профиль/кабинет и сделать систему ролей предсказуемой. Ничего нового — только архитектура.

---

## Роли

| Роль | Видит / может |
|------|-----------------|
| **Гость** | Главная, поиск, карточки. НЕ видит: профиль, сообщения, кабинет, добавление. Бургер есть. |
| **Пользователь (арендатор)** | + нижнее меню, профиль, сообщения, избранное. Искать, писать, бронировать, избранное. |
| **Арендодатель** | + Мои объявления, Бронирования, Финансы, Продвижение. В нижнем меню — «Добавить». |
| **Админ** | + Админ панель (только через /admin и меню аватара). НЕ в профиле. |

---

## Выполнено

### 1. Header

- **Mobile гость:** [бургер] LOCUS [поиск] [колокол]. Кнопка «Войти» только на desktop (xl).
- **Mobile авторизован:** LOCUS [поиск] [колокол] [аватар]. Без бургера, без сообщений в шапке, без отдельной кнопки профиля.
- **Desktop:** LOCUS [поиск] [сообщения] [колокол] [аватар] (или «Войти» для гостя). Сообщения — только на desktop.
- **Клик по аватару:** Профиль, Мои объявления (если landlord), Бронирования (если landlord), Настройки, Админ панель (если admin), Выйти.
- **Бургер (гость):** убран пункт «Тарифы». Оставлены: Поиск жилья, Помощь, Контакты, Войти.

**Файл:** `frontend/components/layout/Header.tsx`

### 2. Нижнее меню (mobile)

- **Арендатор (не landlord):** Главная, Поиск, Сообщения, Избранное, Профиль.
- **Арендодатель (landlord):** Главная, Поиск, Добавить, Сообщения, Профиль.
- **Гость:** те же 5 слотов; Сообщения и Избранное ведут на логин с redirect, последний — «Войти».
- Убрано: тарифы, доходы, дашборд из меню.

**Файл:** `frontend/components/layout/BottomNavGlobal.tsx`

### 3. Профиль (mobile/страница профиля)

- Разделы: Мои объявления (только если landlord), Бронирования, Сообщения, Избранное, Продвижение (если landlord), Финансы (если landlord), Настройки, Выйти.
- Убрано: доходы, аналитика, дашборд, админ из профиля.

**Файл:** `frontend/components/profile/ProfileSidebar.tsx`

### 4. Desktop навигация

- **Header:** LOCUS, поиск, сообщения (иконка), колокол, аватар.
- **Sidebar (авторизован):** Главная, Поиск, Сообщения, Избранное, Профиль. Если landlord: блок «Арендодатель» (Мои объявления, Бронирования, Финансы, Продвижение). Если admin: Админ панель. Внизу — Выйти.
- Убрано из sidebar: отдельная кнопка «Добавить объявление» (есть в header «Сдать жильё»). Блок «Арендодатель» показывается только при isLandlord.

**Файл:** `frontend/components/layout/AppSidebarTZ14.tsx`

### 5. Админ панель

- Доступ: только /admin. Вход: пункт «Админ панель» в меню аватара (если role = admin). Не в профиле, не в нижнем меню.

### 6. Удалённые дубли

- Тарифы убраны из бургер-меню гостя.
- Сообщения не дублируются в header на mobile (только в нижнем меню).
- «Войти» у гостя только на desktop.
- Мои объявления и Бронирования в аватар-меню только для landlord.
- В профиле нет админа, доходов, дашборда.

### 7. Кнопка «Добавить»

- Ведёт в создание объявления (/create-listing), не на тарифы. Тарифы — в конце публикации и в разделе «Продвижение».

---

## Критерии готовности (выполнены)

- Бургер только у гостя.
- Админ доступен через меню аватара и sidebar.
- Дублей нет (профиль/сообщения/тарифы/админ).
- Профиль очищен от лишнего.
- Меню по ролям (арендатор без «Добавить», арендодатель с «Добавить»).
- Desktop совпадает с mobile по логике ролей.

# TZ-22 — Финальная архитектура навигации и структуры LOCUS

## Цель
Единая логичная структура (mobile-first), один центр управления, без дублирующих меню и сломанных переходов.

## Выполнено

### 1. Нижнее меню (mobile) — всегда 5 пунктов
- **Главная** → /
- **Поиск** → /listings
- **Добавить** → не авторизован: логин с redirect; авторизован: /dashboard/listings/create
- **Сообщения** → /messages (или логин)
- **Профиль** → /dashboard (или логин с redirect=/dashboard)

Дубли убраны: один нижний бар, без вариантов по ролям (Избранное перенесено в кабинет).

### 2. Единый кабинет /dashboard
- Экран с блоками: Обзор (профиль, избранное), Мои объявления (список + добавить), Финансы, Продвижение, Настройки профиля.
- Каждый блок ведёт на /dashboard/listings, /dashboard/billing, /dashboard/promo, /dashboard/profile (редиректы на существующие страницы).
- Не авторизован → логин с redirect=/dashboard.

### 3. Desktop — sidebar | content
- Layout уже: AppSidebarTZ14 + main.
- Sidebar приведён к тем же 5 пунктам: Главная, Поиск, Добавить, Сообщения, Профиль. Дополнительно: Админ (если admin), Выйти (если авторизован).

### 4. Header
- Без изменений по ТЗ-22 (оставлен как в TZ-21): [бургер] LOCUS [поиск] [уведомления] [аватар].

### 5. Боковое меню (бургер)
- **Гость:** Поиск, Тарифы, Помощь, Контакты, Войти.
- **Авторизован:** Кабинет, Мои объявления (если landlord), Сообщения, Финансы (landlord), Продвижение (landlord), Настройки, Выйти.

### 6. Фильтры поиска
- FiltersModal уже на BottomSheet. Добавлены: ручка сверху, animateClose, закрытие свайпом вниз по ручке.

### 7. Роли
- Гость: просмотр, поиск.
- Пользователь: кабинет, сообщения, избранное (в кабинете).
- Арендодатель: объявления, финансы, продвижение (в кабинете и сайдбаре).
- Админ: админ-панель в sidebar.

## Не трогали
- AI-метрики, карточка объявления, дизайн (по ТЗ).

## Статус
Финальная навигационная модель зафиксирована. Дальше — доработки и полировка.

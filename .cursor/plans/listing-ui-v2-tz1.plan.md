# ТЗ 1 — Полный редизайн карточки объявления (Listing UI v2)

## Цель
Убрать «пустые» объявления: добавить информативность, быстрые данные, бейджи доверия, мини-галерею, быстрые действия; уровень Циан/Airbnb.

## Реализовано

### Переключение (Part 7)
- **Feature flag:** `useNewListingUI` в `config/uiFlags.ts` и `shared/config/listingUi.ts` (true).
- **components/listing/index.ts:** при `useNewListingUI` экспортируются `ListingCardV2` и `ListingCardV2Skeleton` как `ListingCard` / `ListingCardSkeleton` — все места (HomePageV6, ListingsGridTZ11, SearchOverlay, favorites и т.д.) автоматически используют новый UI.
- **app/listings/[id]/PageClient.tsx:** при `useNewListingUI` рендерится `ListingPageV2TZ1`, иначе `ListingPageLight` (V3).

### ListingCardV2 (Part 1–3, 5)
- **Блок 1:** Фото 16:10, поверх фото: цена, тип (посуточно/мес), рейтинг, бейджи (Проверено документами, PRO, Новое, Суперхозяин, Собственник, Агент).
- **Блок 2:** Основная инфа: N-комн • площадь • этаж; город • метро; район.
- **Блок 3:** Быстрые параметры — сетка (комнаты, площадь, этаж, тип дома) с иконками.
- **Блок 4:** Краткое описание до 140 символов, 2 строки.
- **Блок 5:** Быстрые действия (сохранить, написать, позвонить, забронировать) — визуальные кнопки.
- **Мини-галерея (Part 2):** карусель, до 5 мини-превью, свайп, счётчик фото.
- **Компактный режим (Part 3):** `compact={true}` — высота ≤220px, [фото] + цена, район, параметры, метро, рейтинг.
- **Статусы доверия (Part 5):** типы бейджей в `ListingCardBadge`: verified, top→PRO, new, superhost, owner, agent; подписи в `ListingCardV2` (Проверено документами, Суперхозяин, Собственник, Агент).

### ListingPageV2TZ1 (Part 4)
- **Галерея:** полноэкран, основное фото и мини-галерея.
- **Инфо:** цена, адрес, метро, карта, владелец, рейтинг.
- **AI-блок:** блок «AI анализ» (цена ниже рынка, район безопасный, ликвидность и т.д.) — ключевая фишка.
- **Бейджи доверия:** проверено документами, собственник, агент, супер-хозяин (TRUST_BADGES в компоненте).

### Что не трогаем (Part 6)
- Backend объявлений, API, маршруты, авторизация — без изменений, только UI.

## Проверка (Part 8)
- Карточка: информативная, не пустая, компактная, с максимумом инфы.
- Страница объявления: галерея, инфо, AI-блок, бейджи.
- Отключение: `useNewListingUI = false` в `config/uiFlags.ts` — возврат к старым карточке и странице.

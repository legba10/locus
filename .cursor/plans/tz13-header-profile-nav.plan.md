# ТЗ-13: Хедер, профиль и навигация

## Цель
Чистая навигация без дублей. Одна точка входа в профиль. UX как у Airbnb/Циан.

## Выполнено

### 1. Удалены дубли
- **Один вход в профиль** — только аватар справа (клик → меню профиля).
- Убрана иконка локации (MapPin) из левой части хедера.
- Убрана кнопка «Разместить» из правой части (разгрузка; создание объявления через «Мои объявления» в меню профиля).
- В бургер-меню убраны: Профиль, Мои объявления, Разместить объявление, Выйти. Профиль и выход только через аватар.

### 2. Структура Header
- **Слева:** бургер (только mobile), логотип.
- **Справа:** колокольчик (уведомления), иконки «Сообщения» и «Избранное» (desktop), переключатель темы, кнопка «Войти» или аватар.

### 3. Меню профиля (по клику на аватар)
Пункты: Профиль, Мои объявления, Сообщения, Избранное, Тарифы, Настройки (только mobile в меню), Админ (если админ), разделитель, Выйти.

### 4. Бургер-меню
Только навигация: Поиск жилья, Избранное, Сообщения, Тарифы, Админ (если админ), Помощь. Для гостя сверху — кнопка «Войти / Зарегистрироваться». Переключатель темы только в хедере, в бургер не выносился.

### 5. Визуал и высота
- Высота: 60px (mobile), 64px (desktop).
- Фон и граница: существующие rgba + blur и тонкая граница в `layout-header.css`.
- Иконки через `var(--text-main)` для стабильного отображения в светлой и тёмной теме.

### 6. Состояния
- Гость: справа кнопка «Войти».
- Пользователь: справа аватар, по клику — меню профиля.
- Админ: то же + пункт «Админ» в меню профиля.

## Не трогали
- Логотип (только расположение в сетке).

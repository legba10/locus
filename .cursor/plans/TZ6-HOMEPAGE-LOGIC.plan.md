# ТЗ-6: Полная логика главной страницы (структура, поток, UX)

**Цель:** единый пользовательский поток за 5–7 действий; главная как инструмент подбора, не лендинг.

## Статус: ✅ Выполнено

## Порядок блоков (ТЗ-6)

1. **Экран 1:** Hero + кнопка (скролл к поиску) + быстрый поиск (город, бюджет, тип, Показать варианты, Фильтры).
2. **Экран 2:** Популярные города (отступ сверху 48px).
3. **Экран 3:** Расширенный поиск — карточка: город, бюджет, тип, комнаты, кнопка Показать варианты, Фильтры.
4. **Экран 4:** Лучшие объявления (Актуальные предложения), `id="listings"` — scroll target.
5. **Экран 5:** Один блок AI подбор (карточка «Умный подбор», кнопка открывает модалку).
6. **Экран 6:** Новости рынка (цены, тренды, районы).
7. **Экран 7:** Статистика (StatsBlock).
8. **Экран 8:** Сдать жильё + футер.

## Что сделано

- **Hero:** без AI-карточки под ним; кнопка скроллит к поиску (как в ТЗ-3/ТЗ-5).
- **Быстрый поиск:** сразу под Hero, табы Умный/Ручной, одна карточка (город, бюджет, тип, Показать, Фильтры).
- **Популярные города:** сразу после поиска, класс `home-tz6-cities` — отступ сверху 48px.
- **Расширенный поиск:** новая секция после городов с полями город, бюджет, тип, **комнаты** (ROOMS_OPTIONS), те же кнопки; общий store.
- **Объявления:** заголовок «Актуальные предложения», без дублирования секций.
- **Один AI-блок:** перенесён после объявлений; удалены секции «Рекомендации AI», «Последние просмотренные», MarketAnalysisBlock, HowItWorks, AIBlock.
- **Новости рынка:** перед статистикой.
- **Статистика:** после новостей (почти в конце).
- **Отступы:** класс `home-tz6-block` — 48px (mobile), 56px (desktop); максимум 64px по ТЗ.
- **Sticky поиск:** без изменений (ТЗ-5).
- **Переходы:** «Показать варианты» скроллит к `#listings` и применяет фильтр (без новой страницы).

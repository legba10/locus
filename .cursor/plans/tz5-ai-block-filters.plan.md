# ТЗ-5: Блок «Подберём жильё за 10 секунд» + исправление фильтров

## Цель
Понятный сценарий AI-подбора, без багов (скролл, зависание фильтров), как в Airbnb smart search.

## Сделано

### 1. Блок на главной (HomePageV6)
- Заголовок: «Подберём жильё за 10 секунд»
- Подзаголовок: «AI найдёт лучшие варианты под вас»
- Кнопка: «Начать подбор» — открывает модалку (не переход на поиск)

### 2. AIWizardModal
- **Body scroll:** при открытии `body overflow: hidden`, при закрытии восстанавливается.
- **Скролл только в модалке:** контент в `overflow-y-auto`, футер и шапка не скроллятся.
- **Пошаговые шаги:**
  - Шаг 1: Город («Где ищете?») — CitySelect, подставляется initialCity.
  - Шаг 2: Бюджет — пресеты «до 30 000», «30 000–50 000», «50 000+» (₽/мес).
  - Шаг 3: Тип жилья — Квартира, Студия, Дом, Комната (multi select).
  - Шаг 4: Даты — заезд и выезд (два поля date).
- **Подсказка под кнопкой:** «Обычный поиск — вы настраиваете всё сами. AI-подбор — система сама находит лучшие варианты.»
- Закрытие: крестик, Esc, клик по overlay.
- После «Подобрать» — вызов onComplete и переход на `/listings?ai=true&city=...&priceMin=...&priceMax=...&dateFrom=...&dateTo=...&type=...`.

### 3. Страница поиска (SearchPageV4)
- При `ai=true` сверху показывается блок: «Мы подобрали жильё под вас» + текст про изменение фильтров.

### 4. AIBlock.tsx
- Убран прямой переход на `/listings?ai=true`.
- Текст блока приведён к ТЗ-5.
- Кнопка «Начать подбор» вызывает опциональный `onStartClick` (если передан). Без callback кнопка disabled.

## Acceptance
- [x] Модалка открывается по кнопке
- [x] Фильтры не зависают, скролл только внутри модалки
- [x] Понятна разница AI и обычного поиска (подсказка)
- [x] Mobile: bottom sheet, скролл внутри
- [x] Нет двойного скролла (body lock)
